# Context Refresh — Rebuild Project Context + Page Index

You scan the project's Notion page tree, rebuild the page index, and regenerate the project context file. This is a heavy-read, light-write skill — reads many pages and writes two artifacts: the page index and the context file.

## Input

- `project_slug` — required
- `store_on_notion` — optional boolean; if true, also update the Context Page on Notion (default: true)

## Prerequisites

Use the "Project Configurations" section injected into your system prompt for Notion database IDs and workspace info.

## Execution Protocol

### Phase 1: Scan Page Tree

1. Fetch the project's Notion parent page (from config)
2. List all child pages recursively
3. For each page, record: Page ID, Title, Type (page/database), Parent, Last edited timestamp
4. **Exclude** pages under sections titled "Archive" (case-insensitive)
5. **Exclude** internal bot-state databases (e.g., Pending Interactions, Run History DBs) — these are not user-facing and should not appear in indexes

### Phase 2: Rebuild Page Index

Generate a structured page index:

```markdown
# Notion Page Index — {project_name}

> Rebuilt by context-refresh on {ISO timestamp}
> Scoping: children of {parent_page_name} ({parent_page_id}), excluding Archive + bot-state

## Databases

| Database | Data Source ID | Purpose |
|---|---|---|
| {db_name} | {data_source_id} | {inferred purpose} |

## Key Pages

| Page | ID | Purpose | Last Edited |
|---|---|---|---|
| {page_title} | {page_id} | {inferred purpose} | {last_edited} |

## Partner Pages

| Partner | Page ID | Last Edited |
|---|---|---|
| {partner_name} | {page_id} | {last_edited} |
```

### Phase 3: Generate Context File

Read from Notion to build the context snapshot:

**3a. Task Tracker Summary**
- Query the project's primary database
- Count items by Status (Not Started, In Progress, Blocked, Done, Closed)
- Identify overdue items (Target Date < today, Status not Done/Closed)
- Identify items due this week
- Identify blocked items with blocker descriptions

**3b. Partnership Statuses** (if partners configured)
- For each partner page found in the index, fetch a summary
- Current status, key open items, last update date

**3c. Key Milestones**
- Search for timeline/milestone pages
- Extract key dates and milestones

**3d. Open Decisions**
- Search meeting notes and context pages for unresolved decisions

**3e. Compile**

```markdown
# {project_name} Context File

> Last updated: {ISO timestamp}
> Generated by: context-refresh

## Task Summary

| Status | Count |
|---|---|
| Not Started | {n} |
| In Progress | {n} |
| Blocked | {n} |
| Done | {n} |
| Total Active | {n} |

*Overdue:* {count} items
*Due this week:* {count} items

## Blocked Items

{list with IDs, titles, owners, blocking reason}

## Partnership Statuses

| Partner | Status | Key Items | Last Updated |
|---|---|---|---|
| {name} | {status} | {items} | {date} |

## Key Milestones

{milestones with dates}

## Open Decisions

{unresolved decisions}

## Team Roster

{from project config: names, roles, Slack IDs}

## Notion Page Index

{table of all in-scope pages with IDs}
```

### Phase 4: Store

1. Find or create the Context Page under the project parent on Notion, replace its content with the generated context file
2. Store the page index as a child page under the Context Page on Notion (titled "{Project} — Page Index"). Other skills can fetch this page to resolve page references.

### Phase 5: Report

If triggered by user command (not cron):
- Reply: "Context refreshed. {N} pages indexed. Context file updated."
- Include brief summary of what changed since last refresh (if determinable from timestamps)

## Output Format

```json
{
  "project": "{slug}",
  "pages_indexed": 24,
  "databases_found": 2,
  "partner_pages": 3,
  "overdue_items": 4,
  "blocked_items": 1,
  "context_stored_on_notion": true,
  "timestamp": "2026-03-13T10:00:00Z"
}
```

## Guardrails

- Heavy read, light write. The only writes are: updating the Context Page and Page Index page on Notion. No local storage — all state lives on Notion.
- EXCLUDE internal bot-state databases from user-facing indexes. Bot infrastructure should be invisible.
- EXCLUDE archived pages. They're archived for a reason.
- If Notion API times out: retry once after 10 seconds. If still failing, report the error.
- If a specific page is inaccessible (404/permission): skip it, note in the index, continue.
- NEVER fabricate page data — only index what Notion returns.
